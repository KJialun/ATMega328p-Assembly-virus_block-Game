Netlist and libraries

LTSpice (a variant on Berkeley university's "Spice") is a circuit simulator. It can start from a textual description of the circuit called a netlist. The netlist is a text file (usually with the extension ".net") that is passed to LTSpice, parsed and simulated. However, as a designer it is not that handy to always write a netlist file for each circuit in a text editor, which is why LTspice has a schematic editor. The job of this schematic editor is to allow the designer to draw circuits, which it then converts to a netlist.

You can use the following parameters (warning! sometimes the unit of time 'u'  is automatically changed to 'h' by the browser. be sure to use 'u' in LTspice):

  • Vinitial: 0 (the initial voltage)
  • Von: 5 (the pulse voltage)
  • Tdelay: 1u (delay, 1u means 1 microsecond in Spice)
  • Trise: 1n (rise time, 1n means 1 nanosecond in Spice)

Warning: ALL parameters need to be specified. Due to the nature of Spice netlists, unexpected things can happen if you omit any of these parameters. The general syntax for a pulse statement for a netlist is PULSE([V1] [V2] [TD] [TR] [TF] [PW] [PER]). If you omit any parameter, LTspice will not be able to distinguish the omitted parameter from the others.

The resistor has a value of 1k, which is 1k(ilo Ohm) in Spice. The capacitor has a value of 1n(ano Farad). Many other modifiers like this exist:

  • 1f = 1e-15
  • 1p = 1e-12
  • 1n = 1e-9
  • 1u = 1e-6
  • 1m = 1e-3
  • 1k = 1e3
  • 1meg = 1e6 (Spice is not case-sensitive, so 1M would be 1e-3)
  • 1g = 1e9
  • 1t =1e12

Any characters that follow these modifiers are ignored. For example, 1.5kOhm is treated the same as 1.5kA, 1.5KV or 1.5e3.

This type of analysis is also called a small-signal analysis. It computes the operating point of the circuit first, and then considers small perturbations on top of that operating point that vary with frequency. The analysis then approximates all nonlinear devices by linear ones (using a small-signal equivalent model). This approximation is fine for very small perturbations. For example, an amplifier can be biased around 1V+-1mV, and amplifies it to 5V+-10mV. The small-signal analysis allows you to find the signal gain of 10 (it amplifies 1mV to 10mV). This type of simulation is really fast due to the linearization!

The capacitor

One of the most important elements to CMOS design is identifying and analyzing capacitors. The capacitor has the following relation:

ic=Cdvcdt

Or, the same equation using the total charge stored on the capacitor:

Qc=Cvc

This relation will always hold for ideal capacitors. An ideal capacitor has no limit on the maximum voltage, maximum current, etc. In practice however, a non-ideal capacitor will have a maximum rating. If the voltage across the capacitor gets too large the capacitor will break down. High currents are also not always a good idea, as the heat dissipation may break the circuit.

From the capacitor equation, you can deduce a few interesting properties that are often used:

  1. When a circuit is static, meaning that no voltage or current changes over time, then that means that the capacitor current is exactly 0. The capacitor therefore behaves in this case as an open circuit.
  2. Alternatively, you can also state that the voltage across a capacitor remains continuous. If the voltage would be discontinuous, then the derivative would become infinite, leading to an infinite current (there are some theoretical exceptions).
